An Easy Way To Fine-Tune Stable Diffusion with Dreambooth (Colab)¶

In their awesome paper "DreamBooth: Fine-tuning Text-to-Image Diffusion Models for Subject-Driven Generation", introduced at CVPR (Conference on Computer Vision and Pattern Recognition) 2023 its authors Ruiz N, Li Y, Jampani V, Pritch Y, Rubinstein M, and Aberman K present a novel approach to enhancing text-to-image models through personalized fine-tuning.

It fine-tunes a pretrained model with a small set of subject-specific images, enabling it to associate a unique identifier with the subject. This allows the model to generate novel, photorealistic images of the subject in diverse contexts and conditions, leveraging semantic priors and a specialized preservation loss.

Benefits:

  1. Personalization: Tailors image generation to specific subjects based on a few reference images.
  2. Versatility: Enables synthesis of images in various scenes, poses, views, and lighting conditions not present in the reference set.
  3. Realism: Produces photorealistic outputs that preserve key features of the subject.
  4. Applications: Facilitates subject recontextualization, text-guided view synthesis, appearance modification, and artistic rendering.

This tutorial helps you train with limited GPU vRAM, like Colab, free 15GB GPU tier or on a local machine.

import matplotlib.pyplot as plt
import matplotlib.image as mpimg

def show_images_in_directory(directory):
    fig = plt.figure(figsize=(6, 4))  # Adjust the figure size as needed
    images = [file for file in os.listdir(directory) if file.endswith(('jpg', 'png', 'jpeg'))]  # Filter image files
    num_images = len(images)
    for i, image_name in enumerate(images):
        image_path = os.path.join(directory, image_name)
        img = mpimg.imread(image_path)
        ax = fig.add_subplot(1, num_images, i + 1)
        ax.imshow(img)
        ax.axis('off')
        # ax.set_title(image_name)
    plt.tight_layout()
    plt.show()
show_images_in_directory(INSTANCE_DIR)

Step 4: Run the Training¶

Tips:

  1. Low learning rate and progressively increasing the number of steps.
  • High learning rates and too many training steps will lead to overfitting. The model will mostly generate images from your training data, no matter what prompt is used.
  • Low learning rates and too few steps will lead to underfitting: the model will not be able to generate the concept we were trying to incorporate.
  • A learning rate of 2e-6 with 400 training steps works well for objects.
  1. For faces, 800-1200 steps worked well when using a batch size of 2 and LR of 1e-6.
  2. Prior preservation is important to avoid overfitting when training on faces. For other subjects, it doesn't seem to make a huge difference.
  3. If images are still noisy run inference for 100 steps.
  4. Training the text encoder in addition to the UNet has a big impact on quality. Using techniques like 8-bit Adam, fp16 training or gradient accumulation reduces memory requirements to run on 15 GB GPUs like the ones provided by Google Colab.
  5. Try different schedulers during inference process, i.e. PNDM, DDIM, LMSDiscrete etc.

from diffusers import StableDiffusionPipeline
import torch

model_id = "/content/dreambooth/weights_dir"
pipe = StableDiffusionPipeline.from_pretrained(model_id, torch_dtype=torch.float16).to("cuda")

prompt = "A photo of rc_car on a beach"
image = pipe(prompt, num_inference_steps=50, guidance_scale=7.5).images[0]
display(image)
